# piko TUI — 产品需求文档 (PRD)

> 基于 `packages/host-tui/docs/` 现有设计文档提炼的功能需求，刨除技术实现细节。
> 版本：2026-06-29

---

## 1. 产品概述

piko TUI 是 piko 编码智能体（coding agent）的终端用户界面。它提供：

- **对话式时间线**：用户与 AI 的多轮对话、工具调用、思考内容，以结构化的时间线呈现。
- **多智能体可视化**：在团队模式下，同时展示多个 AI 智能体的运行状态、任务计划和队列。
- **内联操作面板**：模型选择、推理深度设置、会话管理、历史通知等操作以嵌入式面板呈现，不打断工作流。
- **完全键盘驱动**：所有操作均可通过键盘完成，支持自定义键位绑定。

---

## 2. 视口布局与区域划分

终端视口被划分为固定的插槽（slot），从上到下依次为：

```
┌─────────────────────────────────────┐
│  时间线 (Timeline)                   │  ← 主内容区：对话消息、工具调用、思考内容
├─────────────────────────────────────┤
│  状态栏 (Status)                     │  ← 智能体面板 + 通知行
├─────────────────────────────────────┤
│  编辑区 (Editor)                     │  ← 用户输入框
├─────────────────────────────────────┤
│  底栏 (Bottom Bar)                   │  ← 会话名、模型、用量、cwd、git 分支
└─────────────────────────────────────┘
```

**基本规则：**

| 需求编号 | 需求描述 |
|----------|----------|
| **L1** | 时间线占据剩余垂直空间，高度由视口减去状态栏、编辑区、底栏后自动计算。 |
| **L2** | 状态栏高度由其内容决定（`flexShrink=0`），不占用额外空间。 |
| **L3** | 编辑区高度固定或由内容决定。 |
| **L4** | 底栏始终渲染，不被任何面板替换。 |
| **L5** | 当有捕获式面板（capture panel）激活时，状态栏和编辑区从布局中移除，面板占据剩余空间。 |
| **L6** | 窄终端（宽度 < 一定阈值）时，部分内容自动压缩或隐藏次要信息。 |

## 3. 键盘交互与焦点系统

piko TUI 是键盘优先的界面。所有按键事件通过统一路由分发。

### 3.1 按键优先级

| 需求编号 | 需求描述 |
|----------|----------|
| **K1** | 全局 Esc 键优先处理：关闭面板 → 取消自动补全 → 中断运行中的流 → 双击 Esc 打开会话树/fork。 |
| **K2** | Enter 键优先级：面板确认/提交 → 自动补全接受 → 编辑器提交 prompt。 |
| **K3** | 可打印字符优先级：捕获式面板输入 → 编辑器草稿。 |
| **K4** | 当捕获式面板激活时，编辑器不在渲染树中，结构上无法接收输入（不仅靠焦点判断）。 |
| **K5** | 当阻塞式面板激活时，全局快捷键（如 Ctrl+L、Ctrl+P）被屏蔽。 |
| **K6** | PageUp / PageDown / End 在编辑器获得焦点且无阻塞面板时，应控制时间线滚动。 |
| **K7** | 自动补全的导航键（↑/↓/Tab/Enter/Esc）在编辑器子处理器中处理，不经过完整焦点路由。 |

### 3.2 焦点管理

| 需求编号 | 需求描述 |
|----------|----------|
| **K8** | 编辑器为默认焦点所有者。 |
| **K9** | 打开捕获式面板时，焦点推入面板，关闭时恢复到编辑器。 |
| **K10** | 双击 Esc 行为可配置：打开会话树 / fork 会话 / 无操作。 |

### 3.3 Esc 键完整优先级

| 需求编号 | 需求描述 |
|----------|----------|
| **K11** | 当工具审批待处理时：Enter = 接受，Esc = 拒绝（阻塞所有其他按键）。 |
| **K12** | 当有激活的表面面板：关闭顶部面板。 |
| **K13** | 当自动补全可见：取消自动补全。 |
| **K14** | 当流正在运行：中断流。 |
| **K15** | 编辑器为空时双击 Esc：执行配置的双击动作（会话树/fork/无）。 |

## 4. 面板与表面系统

面板（Panel）以表面（Surface）的形式呈现。表面是覆盖在基础布局之上的 UI 单元。

### 4.1 表面类型

| 需求编号 | 需求描述 |
|----------|----------|
| **S1** | 支持 **完整式（full）** 面板：替换时间线区域，占据所有剩余垂直空间。用于：会话恢复、会话树（窄终端）、通知历史、登录/设置向导。 |
| **S2** | 支持 **插入式（partial）** 面板：插入在状态栏和编辑区之间。用于：模型选择器、推理深度选择器、设置页、重命名、审批确认。 |
| **S3** | 支持 **捕获式（capture）** 输入策略：阻止按键到达编辑器。用于所有交互式面板。 |
| **S4** | 支持 **被动式（passive）** 输入策略：不阻止编辑器输入。用于只读状态面板。 |

### 4.2 表面生命周期

| 需求编号 | 需求描述 |
|----------|----------|
| **S5** | 同时只能有一个 full 面板和一个 partial 面板可见（各自取最高 z-order）。 |
| **S6** | 关闭一个面板时，其所有子面板同时关闭。 |
| **S7** | z-order 递增分配（每次打开 +10），子面板 z-order 高于父面板。 |
| **S8** | 面板支持两种关闭策略：路由栈为空时自动关闭（`route-pop-or-close`）/ 仅手动关闭（`manual`）。 |

### 4.3 面板交互模式

| 需求编号 | 需求描述 |
|----------|----------|
| **S9** | **列表（list）**：↑/↓/PageUp/PageDown 导航，Enter 确认选择，Esc 关闭。可选支持过滤。 |
| **S10** | **菜单（menu）**：与列表相同，Enter 激活选项。可选支持过滤。 |
| **S11** | **表单（form）**：文本输入，Tab/Shift+Tab 切换字段，Enter 提交，Esc 取消。 |
| **S12** | **确认（confirm）**：←/→ / Tab 选择，Enter 确认，Esc 取消。用于是/否或破坏性操作确认。 |
| **S13** | **被动（passive）**：不捕获按键，用于只读状态面板。 |

### 4.4 面板路由

| 需求编号 | 需求描述 |
|----------|----------|
| **S14** | 面板支持路由栈（push/pop/replace），支持嵌套导航（如设置 → 模型列表 → 提供商列表）。 |
| **S15** | 面板有标题栏（chrome.title）和提示行（chrome.hints）。 |
| **S16** | 提示行文本来自面板交互类型 × 键位绑定，不使用硬编码字符串。 |

### 4.5 各面板功能清单

| 功能 | 命令 | 放置方式 | 输入策略 | 说明 |
|------|------|----------|----------|------|
| 模型选择器 | `/model`、`Ctrl+L` | partial | capture | 列出可用模型，支持过滤和切换 |
| 推理深度选择器 | `/thinking` | partial | capture | off/low/medium/high 四级选择 |
| 设置 | `/settings` | partial | capture | 多级菜单，含模型、主题、思考等子页 |
| 会话恢复 | `/resume` | full | capture | 显示最近会话列表供恢复 |
| 会话树 | `/tree`、双击 Esc | 窄屏 full / 宽屏 partial | capture | 平铺显示会话树，支持导航 |
| 通知历史 | `/notifications`、`/noti` | full | capture | 按时间倒序显示通知，支持展开详情 |
| 登录 | `/login` | partial | capture | API 密钥配置 |
| fork 会话 | `/fork` | partial | capture | 确认式面板 |
| 会话重命名 | `/rename` | partial | capture | 表单式面板 |
| 工具审批 | 自动触发 | partial | capture | 阻断编辑器直到审批完成 |

## 6. 时间线与消息系统

### 6.1 消息类型与呈现

| 需求编号 | 需求描述 |
|----------|----------|
| **T1** | 展示用户消息：视觉上与 AI 消息区分，保留多行内容。 |
| **T2** | 展示 AI 助手消息：纯文本可读，流式输出时显示闪烁光标。 |
| **T3** | 展示工具调用/结果：显示工具名称、状态（等待/运行中/成功/错误）、摘要。长的工具结果默认折叠。 |
| **T4** | 展示思考内容：低调的可展开区域，受 `hideThinking` 设置控制。 |
| **T5** | 展示分支/压缩摘要：克制的视觉强调，与普通消息视觉分离。 |
| **T6** | 展示审批结果：审批完成后内联显示结果。 |
| **T7** | 展示系统注释：仅当属于对话上下文时才显示会话生命周期注释。 |
| **T8** | 消息以确定性顺序排列（消息在前，工具紧随其父消息，按 toolCallIndex 排序），不依赖事件到达时间。 |

### 6.2 流式行为

| 需求编号 | 需求描述 |
|----------|----------|
| **T9** | 用户提交 prompt 后立即在时间线末尾创建用户消息条目。 |
| **T10** | 开始流式输出时创建助理消息条目，后续文本增量更新同一条目（不新建）。 |
| **T11** | 流式结束时助理消息条目确定化，视觉上不跳跃。 |
| **T12** | 工具调用在流式过程中插入到父消息之后。 |

### 6.3 滚动行为

| 需求编号 | 需求描述 |
|----------|----------|
| **T13** | 默认跟随底部自动滚动。 |
| **T14** | 用户向上滚动后进入手动模式，不再自动跟随；显示待查看新消息计数。 |
| **T15** | 用户滚回底部时恢复自动跟随，清除待查看计数。 |
| **T16** | 提供"跳转最新"快捷键恢复底部锚点。 |
| **T17** | PageUp/PageDown/End 通过编辑器拦截器调度滚动命令。 |

## 7. 多智能体可视化

### 7.1 智能体状态

| 需求编号 | 需求描述 |
|----------|----------|
| **A1** | 团队模式下，显示所有活跃智能体的运行状态。 |
| **A2** | 每个智能体有状态：空闲（idle）、运行中（running）、失败（failed）、已停止（stopped）。 |
| **A3** | 运行中的智能体显示动画旋转指示器。 |
| **A4** | 失败的智能体显示错误图标和错误信息。 |

### 7.2 智能体面板模式

| 需求编号 | 需求描述 |
|----------|----------|
| **A5** | **折叠模式（collapsed）**：单行显示智能体图标 + 名称 + 计划进度 + 详细信息 + 队列计数。 |
| **A6** | **展开模式（expanded）**：显示智能体头部行 + 计划步骤列表 + 错误行（如有）+ 队列项。头部行仅显示名称（不重复任务标题和队列计数）。 |
| **A7** | 同一时刻只有一个智能体可展开；切换查看的智能体时自动收起之前展开的智能体。 |
| **A8** | 通过 `Ctrl+E` 或点击当前智能体切换展开/折叠。 |

### 7.3 计划步骤

| 需求编号 | 需求描述 |
|----------|----------|
| **A9** | 计划步骤显示在展开模式中，包含：状态图标（● 进行中 / ✓ 已完成 / ○ 待处理）+ 步骤编号 + 步骤描述。 |
| **A10** | 已完成步骤使用成功色（绿色），进行中使用强调色，待处理使用暗淡色。 |

### 7.4 队列显示

| 需求编号 | 需求描述 |
|----------|----------|
| **A11** | 折叠模式：在队列列显示队列计数（如"1 queued"），仅终端宽度 ≥ 64 字符时显示。 |
| **A12** | 展开模式：队列项以独立行显示（↳ 图标 + 类型 + 预览），头部行不显示队列计数。 |
| **A13** | 队列类型：steering（引导）、follow-up（追问）、next-turn（下一轮）。 |

### 7.5 多智能体布局规则

| 需求编号 | 需求描述 |
|----------|----------|
| **A14** | 被查看的智能体（viewedAgentId）使用强调色 + 实心圆点图标。 |
| **A15** | 背景（非当前查看）的智能体使用暗淡色 + 空心圆点。 |
| **A16** | 当团队模式关闭时，显示单一"main"智能体，状态从流状态推断。 |
| **A17** | 所有智能体的排列顺序由上层决定（如快照顺序）。 |

## 8. 通知系统

### 8.1 通知模型

| 需求编号 | 需求描述 |
|----------|----------|
| **N1** | 通知具有严重级别：信息（info）、成功（success）、警告（warning）、错误（error）。 |
| **N2** | 通知具有来源分类：运行时、引擎、工具、命令、认证、模型、会话、扩展、UI。 |
| **N3** | 通知包含消息正文和可选的详细信息。 |
| **N4** | 通知有创建时间和可选的 TTL（生存时间）。 |
| **N5** | 重复通知通过去重键自动去重。 |
| **N6** | 内存中保留最近 200 条通知。 |

### 8.2 通知显示

| 需求编号 | 需求描述 |
|----------|----------|
| **N7** | 状态栏在空闲时显示最新一条未读未过期的通知。 |
| **N8** | 通知行与智能体面板共享相同的左边距对齐（`│` 竖线图标 + 消息文本，按严重级别着色）。 |
| **N9** | 不显示严重级别标签，仅通过颜色区分：错误=红色，警告=黄色，成功=绿色，信息=强调色。 |
| **N10** | 通知在 TTL 到期后自动从状态栏消失（保留在历史中直到超出上限）。 |
| **N11** | 流式输出期间不显示通知（状态栏由流进度占据）。 |
| **N12** | 通知可标记已读，标记后从状态栏移除。 |

### 8.3 通知历史

| 需求编号 | 需求描述 |
|----------|----------|
| **N13** | `/notifications`（别名 `/noti`）打开通知历史面板。 |
| **N14** | 通知按最新优先排序。 |
| **N15** | 后续支持按严重级别/来源过滤。 |
| **N16** | Enter 展开通知详情。 |
| **N17** | 支持清除通知（`c` 键）。 |

### 8.4 通知生产者

| 需求编号 | 需求描述 |
|----------|----------|
| **N18** | 以下场景应产生通知：未知斜杠命令、命令不可用（智能体运行中）、模型切换成功/失败、登录/认证成功/失败、工具审批待处理/被拒、工具执行错误、运行时关闭/恢复错误、会话恢复/导入/导出成功/失败、设置/主题加载错误、扩展状态/错误消息。 |
| **N19** | 以下场景不应使用通知：正常 token 用量、模型名称、cwd、git 分支等稳定状态数据（这些属于底栏）。 |
